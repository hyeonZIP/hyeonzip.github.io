## 2. OSI 참조 모델
### 물리 계층
+ 전송에 필요한 두 장치간의 `실제 접속`과 절단 등 기계적, 전기적, 기능적, 절차적 특성에 대한 규칙 정의 

### 데이터 링크 계층
+ 두 개의 `인접한` 개방 시스템들 간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 `시스템 간 연결 설정과 유지 및 종료` 담당

### 네트워크 계층
+ 개방 시스템들 간의 `네트워크 연결을 관리`하는 기능과 데이터의 교환 및 중계 기능

### 전송 계층
+ 논리적 안정과 균일한 `데이터 전송` 서비스를 제공함으로써 `종단 시스템(End-to-End)` 간에 투명한 데이터 전송 가능

### 세션 계층
+ 송수신 측 간의 `관련성을 유지`하고 대화 제어 담당

### 표현 계층
+ `서로 다른 데이터 표현` 형태를 갖는 시스템 간의 상호 접속을 위해 필요한 계층
+ 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색 수행

### 응용 계층
+ 사용자(응용 프로그램)가 OSI 환경에 접근할 수 있도록 서비스 제공

## 3. 네트워크 관련 장비

### 라우터
+ LAN과 LAN을 연결하는 기능인 브리지에서 데이터 전송의 `최적 경로` 선택 기능 추가

### 리피터
+ 디지털 신호의 장거리 전송을 위해 수신한 신호를 재생 및 출력 전압을 높여 전송
+ ~~마인크래프트의 레스스톤 리피터와 동일하다~~

### 허브
+ 각각의 회선을 `통합 관리`한다
+ 신호 증폭 기능을 하는 리피터의 역할을 포함한다
+ ~~쿠팡 물류 허브~~

### 브리지
+ `LAN과 LAN을 연결`하거나 LAN 안에서의 컴퓨터 그룹을 연결하는 장치
+ 브리지가 n개 일 때, n(n-1)/2개

### 게이트웨이
+ OSI 전 계층의 `프로토콜 구조가 다른 네트워크를 연결하는 장치`
+ LAN에서 다른 네트워크에 데이터를 보내거나 다른 네트워크로부터 데이터를 받아들이는 `출입구` 역할

## 4. 패킷 교환 방식⭐
+ 메세지를 일정한 길이의 패킷으로 잘라서 전송하는 방식

### 가상 회선 방식⭐ [24.06]
+ 논리적인 가상 통신 회선을 `미리 설정`
+ 패킷을 `순서적`으로 운반
+ `정보 전송 전에 제어 패킷에 의해 경로 설정`
+ 모든 패킷은 같은 경로로 `발생 순서대로 전송`

### 데이터그램 방식⭐ [24.06]
+ `연결 경로 설정하지 않음`
+ `순서에 상관없이 독립적`으로 운반
+ 패킷마다 `전송 경로 다름`
+ 패킷은 `목적지의 완전한 주소를 가짐`
+ `순서에 상관없이` 여러 경로를 통해 도착한 패킷들은 수신 측에서 `순서 재정리`

## 5. NAT
+ Network Address Translation 네트워크 주소 변환
+ `한 개의 정식 IP 주소`에 `대량의 가상 사설 IP 주소`를 할당 및 연결하는 기능
+ IP 마스커레이드

## 6. 경로 제어
+ Routing 은 송수신 측 간의 전송 경로 중에서 최적 패킷 교환 경로를 결정하는 기술
+ 대표적인 경로 제어 프로토콜 : IGP(RIP, OSPF), EGP, BGP

## 7. IGP⭐
+ Interior Gateway Protocol 내부 게이트웨이 프로토콜
+ 하나의 자율 시스템(AS) 내의 라우팅에 사용되는 프로토콜

### RIP⭐ [24.06]
+ Routing Information Protocol
+ `거리 벡터 프로토콜`
+ `Bellman-Ford` 알고리즘 사용
+ 소규모 동종의 네트워크 내에서 효율적

### OSPF⭐ [24.02]
+ Open Shortest Path First 
+ `RIP의 단점을 해결`
+ `Dijkstra (다익스트라)` 알고리즘 사용
+ 대규모 네트워크에서 많이 사용

## 8. EGP / BGP
### EGP
+ Exterior Gateway Protocol
+ 자율 시스템 간의 라우팅, 즉 게이트웨이 간의 라우팅에 사용되는 프로토콜

### BGP
+ Border Gateway Protocol
+ `EGP의 단점 보완`

## 9. 페이지 교체 알고리즘⭐

### OPT
+ OPTimal replacement, 최적 교체
+ `앞으로` 가장 오랫동안 사용하지 않을 페이지를 교체
+ 페이지 부재 횟수가 `가장 적게 발생`하는 가장 효율적인 알고리즘

### FIFO
+ First In First Out
+ 가장 먼저 들어와서 가장 오래 있었던 페이지를 교체하는 방법

### LRU⭐ [24.02]
+ Least `Recently` Used
+ `최근에` 가장 오랫동안 사용하지 않은 페이지를 교체

### LFU⭐ [24.02]
+ Least `Frequently` Used
+ 사용 `빈도`가 가장 적은 페이지를 교체

## 10. 비선점 스케줄링
+ 일괄 처리 방식에 적합
+ 종류 : FCFS, SJF, 우선순위, HRN, 기한부 등

### FCFS
+ First Come First Service
+ 도착 순서대로

### SJF
+ Shortest Job First
+ 실행시간이 짧은 프로세스 부터

### HRN
+ Highest Response-ratio Next
+ 우선순위 계산식을 통해 높은 것부터 
+ 우선순위 = (대기 시간 + 서비스 시간) / 서비스 시간

## 11. 선점 스케줄링
+ 빠른 응답 시간을 요구하는 대화식 시분할 시스템
+ 종류 : Round Robin, SRT, 선점 우선순위, 다단계 큐, 다단계 피드백 큐 등

### RR
+ Round Robin
+ 할당된 시간 만큼 실행한 후 다음 프로세스에게 넘김

### SRT
+ Shortest Remaining Time
+ 현재 실행중인 프로세스의 남은 시간과 새로 도착한 프로세스의 실행 시간을 비교
+ 둘 중 낮은 실행시간을 가진 프로세스로 실행
